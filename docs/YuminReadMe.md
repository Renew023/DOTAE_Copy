## 🟩 1주차 (Ver.0.1)

## 📊 01. 캐릭터 능력치 및 데이터 테이블 구조

### 구현 내용
- **켄시(Kenshi)**, **굶지마(Don’t Starve)** 스타일의 방대한 캐릭터 데이터를 설계
- `ScriptableObject` 기반의 임시 데이터 구성 후, **Json 기반 구조로 리팩토링**

### 구현 과정
- `CharacterData` → `CharacterObject` 간 가변형 데이터 전달 방식 설계
- 능력치 가중치를 분리해 테스트 구조 간소화
- 무게, 장착, 아이템 획득 등 다양한 기능 포함
- `DataManager` 중심의 Json 로딩 구조 작성  
  → ID 기반 경로 연결 / 제네릭 클래스 구조화

### 트러블 슈팅
- Excel 데이터를 ScriptableObject처럼 불러오는 데 혼란  
  → ID 기반 접근 구조로 전환하여 해결

### 문제점
- 능력치를 개별 변수로 구성했으나, `List<Stat>` → `Dictionary<StatType, float>` 구조로 리팩토링 필요  -> 해결(ver.3 기준)
- 일부 능력치 미반영 상태 -> 해결(ver.3 기준)

## ⚙️ 02. 2D 플레이어와 몬스터 상태머신

### 구현 내용
- 상태 패턴(State Pattern) 기반의 FSM(Finite State Machine) 구현
- `Input System` 기반 키 입력 처리

### 구현 과정
- `BaseState` → 각 `State`의 공통 로직을 virtual 구조로 구성
- 공격 상태 등은 `BaseState`에서 처리
- `Input System` 분리 설계 (PlayerKey, UI, Item 등)

### 트러블 슈팅
- 현재까지 특별한 이슈 없음

### 문제점
- 에너미 FSM 미구현 -> 해결(ver.2 기준)
- `PlayerKey` 관련 입력 문제로 상태 해제가 제대로 이뤄지지 않음 (추가 디버깅 필요)  -> 해결(ver.2 기준)

## 🎯 12. 스킬 만들기

### 구현 내용
- 플레이어, 에너미, NPC 등이 사용하는 범용 스킬 시스템 구현
- 스킬은 작고 독립된 액션 단위(텔레포트, 대쉬 등)로 구성
- 액션을 조합하여 하나의 스킬로 순차 실행되도록 설계

### 구현 과정
- `Skill → SkillPool → SkillActionBase` 구조로 설계
- `SkillActionBase`: 개별 액션 구현 (부모 → 자식 구조)
- `SkillPool`: 복수 액션을 묶어 순차 실행
- `Skill`: SkillPool 실행 완료 시 자동 종료

### 트러블 슈팅
- `transform.position` 사용 시 캐릭터가 원위치로 복귀되는 문제 발생  
  → `Rigidbody2D.MovePosition` 사용으로 해결

### 문제점
- 스킬 구조 전체적으로 정리가 덜 된 느낌 → 리팩토링 예정

- 일반 몬스터는 **마을 외곽 지역**에 하나씩 생성되어야 함

### 구현 과정
- `PlayerArea`를 활용해 **플레이어 주변에 몬스터가 생성**되도록 구현  
- 초기 스폰 외에도 **각 플레이어가 개별적으로 생성 로직을 실행**함  
  → `MasterClient`가 일괄 생성하지 않는 구조

### 트러블 슈팅
- 특정 일자에 몬스터 습격을 구현하기 위해 **Coroutine 루프 사용 여부 고민**  
  → `yield return`을 적절히 사용하면 수천 개가 아닌 이상 성능 영향이 크지 않다는 판단으로 코루틴 사용 결정
- 6~7주차 몬스터는 스폰될 때 바닥이 Ground인지 비교하지 않았더니, 플레이어가 외곽에 갔을 때 NavMesh 밖에 몬스터가 생김.
  -> LayerMask를 추가하여 Ground를 체크하는 코드를 넣음으로써 해결.

### 문제점
- 현재까지 특별한 문제 없음
